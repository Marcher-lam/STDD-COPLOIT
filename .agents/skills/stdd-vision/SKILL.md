# STDD 愿景文档向导 (/stdd-vision)

## 目标
创建项目的 **Vision Document**（愿景文档），作为 AI 代理进行架构决策和实现选择的真理来源 (Source of Truth)。

---

## 为什么需要愿景文档？

AI 代理虽然强大，但需要上下文才能做出好的决策。没有清晰的愿景：

- 代理可能做出与目标冲突的架构选择
- 功能可能以不符合产品方向的方式实现
- 不同代理可能采取不一致的方法

---

## 愿景文档模板

创建 `vision.md` 文件在项目根目录：

```markdown
# 项目愿景

## 概述

简要描述项目做什么、为谁做。

## 目标

- 主要目标 1
- 主要目标 2
- ...

## 非目标 (超出范围)

这个项目明确不会做的事情。

## 关键约束

- 技术约束（如必须使用特定框架）
- 业务约束（如必须符合 GDPR）
- 性能要求
- 安全要求

## 架构决策

重大技术决策及其理由。

## 用户体验原则

产品应该给用户什么感觉。

## 成功标准

如何衡量项目是否成功。

## 技术栈

- 前端: ...
- 后端: ...
- 数据库: ...
- 部署: ...
```

---

## 步骤

1. **检查现有愿景**
   - 读取项目根目录的 `vision.md`
   - 如果存在，询问用户是更新还是重新创建

2. **收集项目信息**
   - 读取 `.stdd/memory/foundation.md` 获取技术栈信息
   - 读取 `.stdd/memory/components.md` 获取组件架构
   - 读取 `.stdd/memory/contracts.md` 获取接口契约

3. **交互式问答**
   - 如果信息不足，向用户提问：
     - "项目的核心目标是什么？"
     - "有哪些明确的非目标？"
     - "有哪些关键约束？"
     - "用户体验应该是什么样的？"

4. **生成愿景文档**
   - 基于收集的信息生成 `vision.md`
   - 在 `.stdd/memory/vision.md` 也保存一份副本

5. **注册到 AGENTS.md**
   - 在 `AGENTS.md` 中添加愿景文档引用
   - 确保 AI 代理首先读取愿景文档

---

## AI 代理如何使用愿景文档

STDD 系统指导 AI 代理：

1. **首先读取愿景** - 在创建或运行任何任务前，代理读取 `vision.md` 理解项目上下文
2. **检查冲突** - 当给定任务时，代理将其与愿景比较，识别潜在冲突
3. **请求澄清** - 如果任务与愿景冲突，代理会停下来询问如何解决冲突，而不是盲目继续

## 愿景文档与其他 STDD 文件的关系

```
vision.md (项目级真理)
    ↓ 指导
foundation.md (技术栈约束)
components.md (组件架构)
contracts.md (接口契约)
    ↓ 输入
01_proposal.md (当前需求)
    ↓
02_bdd_specs.feature
    ↓
04_tasks.md
    ↓
实现
```

## 示例

### 输入

```
/stdd-vision

项目是一个团队协作工具，目标是让远程团队更高效地协作。
技术栈：React + Node.js + PostgreSQL
约束：必须支持离线使用、数据端到端加密
```

### 输出 (vision.md)

```markdown
# TeamFlow 项目愿景

## 概述

TeamFlow 是一个为远程团队设计的协作工具，帮助分布式团队更高效地沟通和协作。

## 目标

- 提供实时协作功能（文档、白板、任务）
- 支持离线使用，网络恢复后自动同步
- 保证数据安全，端到端加密

## 非目标

- 不支持视频会议（使用 Zoom/Teams 等现有工具）
- 不做项目管理（不做甘特图、资源分配）
- 不做代码仓库管理

## 关键约束

- 技术约束：React 前端、Node.js 后端、PostgreSQL 数据库
- 安全约束：所有数据端到端加密
- 性能约束：支持 100+ 用户同时在线协作
- 离线约束：核心功能必须支持离线使用

## 架构决策

- 采用 CRDT 实现离线协作同步
- 使用 WebSocket 实现实时通信
- IndexedDB 作为离线存储

## 用户体验原则

- 简洁：减少点击，快速完成任务
- 响应：操作即时反馈
- 可靠：离线也能工作

## 成功标准

- 日活跃用户 80% 以上
- 平均任务完成时间减少 30%
- 离线使用率 40% 以上

## 技术栈

- 前端: React + TypeScript + Tailwind CSS
- 后端: Node.js + Express + WebSocket
- 数据库: PostgreSQL (主库) + Redis (缓存)
- 离线存储: IndexedDB
```

---

> **调用方式**：在项目初期运行 `/stdd-vision` 创建愿景文档，或随时运行更新。
