# /stdd:execute - TDD 执行循环

启动严格的防跑偏 TDD 执行闭环 (The Ralph Loop)。

## 使用方式

```
/stdd:execute                  # 执行全部待办任务
/stdd:execute --task=TASK-001  # 执行特定任务
/stdd:execute --next           # 执行下一个任务
```

## Ralph Loop

```
┌──────────────────────────────────────────────────────┐
│                    Ralph Loop                         │
│                                                       │
│  🔴 红灯      →    🔍 静态检查    →    🟢 绿灯       │
│  生成失败测试       语法/类型检查       最简实现       │
│                                                       │
│       →    🧪 伪变异审查    →    🔵 重构    →    ✅   │
│           检测骗绿灯断言       优化代码                  │
│                                                       │
│  ⚠️ 连续失败 3 次 → 自动熔断 (/stdd:revert)           │
└──────────────────────────────────────────────────────┘
```

## 执行流程

1. **聚光灯隔离** - 从 `tasks.md` 取出单个微任务，读取 `contracts.md` 校准接口
2. **Red 阶段** - 生成必然失败的测试，禁止写被测功能实现
3. **静态质检** - 运行语法/类型检查（如 `tsc --noEmit`）
4. **Green 阶段** - 只写让测试通过的最简代码，禁止超纲
5. **变异审查** - 检测骗绿灯断言（如 `expect(x).toBeTruthy()`），打回重作
6. **Refactor** - 优化代码结构
7. **更新前进** - 打勾 `[x]`，继续下一个任务

## 熔断机制

- 连续失败 3 次自动停止
- 提示用户人工判断或手动 `git stash` 回退
- 防止 AI 在错误方向上越走越远
