# STDD 结构化规划模式 (/stdd:prp)

## 目标
使用 **PRP (Plan-Reason-Pattern)** 模式进行结构化规划，确保需求清晰、目标明确、路径可行。

---

## PRP 四要素

| 要素 | 问题 | 输出 |
|------|------|------|
| **What** | 做什么？ | 功能描述、用户故事 |
| **Why** | 为什么？ | 业务价值、问题背景 |
| **How** | 怎么做？ | 技术方案、实现路径 |
| **Success** | 成功标准？ | 验收条件、KPI 指标 |

---

## 步骤

### 1. 收集 What (功能定义)
从用户输入中提取核心功能需求：
```
输入: "实现一个支持 Markdown 导出的 todo-list"

提取:
- 功能: Todo List 管理
- 功能: Markdown 导出
- 功能: 持久化存储
```

### 2. 分析 Why (价值分析)
深入理解业务价值和问题背景：
```
问题: 用户需要记录任务并分享
价值: 提高个人效率，便于团队协作
背景: 现有工具不支持 Markdown 格式导出
```

### 3. 规划 How (技术路径)
设计技术实现方案：
```
技术栈:
- 前端: React + TypeScript
- 存储: localStorage
- 导出: 自定义 Markdown 生成器

架构:
- TodoList 组件 (展示层)
- TodoService (业务逻辑层)
- StorageAdapter (数据层)
- MarkdownExporter (导出层)
```

### 4. 定义 Success (成功标准)
设定可量化的验收条件：
```
功能验收:
- ✅ 用户可以添加/删除/完成 Todo
- ✅ 数据自动保存到 localStorage
- ✅ 一键导出为 Markdown 文件

质量验收:
- 📊 测试覆盖率 >= 90%
- 📊 代码复杂度 <= 10
- 📊 无 TypeScript 错误

性能验收:
- ⚡ 页面加载 < 1s
- ⚡ 操作响应 < 100ms
```

## 使用方式

### 从需求创建 PRP
```bash
/stdd:prp 实现一个支持 Markdown 导出的 todo-list
```

### 交互式完善
系统会逐项询问：
```
> [PRP] 请确认 What - 功能描述是否完整？
> 用户: 增加标签分类功能

> [PRP] 请确认 Why - 业务价值是否清晰？
> 用户: 是

> [PRP] 请确认 How - 技术方案是否可行？
> 用户: 改用 IndexedDB 替代 localStorage
```

### 验证 PRP
```bash
/stdd:prp validate
```

### 确认 PRP
```bash
/stdd:prp confirm
```

---

## PRP 与后续流程的关系

```
/stdd:prp
    ↓ 生成 00_prp.md
/stdd:clarify
    ↓ 基于 PRP 进行澄清
/stdd:spec
    ↓ 基于 PRP 生成 BDD 规格
/stdd:plan
    ↓ 基于 PRP 拆解任务
/stdd:execute
    ↓ 基于 Success 标准验证
```

### 输入

